# Template description textuelle d'un scénario

> Un **scénario** = **une instance de cas d'utilisation** = **valeurs fixées** à l'avance, aucune donnée variable = **un test**. **Pas d'enchaînements alternatif/exceptions**

## Header

- Nom du scénario : ex: *"Le client John Doe retire 200 EUROS alors que son compte n'est crédité que de 150 EUROS"*
- Système concerné :
- Description : Une description succincte du scénario. ex: *"Détaille les étapes lors d'une tentative de retrait avec un crédit insuffisant."*
- Date de création :
- Date de dernière mise à jour :
- Version : 1.0
- Responsable(s) : qui est autorisé à modifier ce document
- Acteur principal :
- Acteurs secondaires :
- Pré-conditions : état du système au démarrage, hypothèses

## Enchaînement

1. Étape 1. *Le système...*
2. Étape 2. *Le système...*
3. Etc.
4. Etc.

## Post-conditions

État du système *attendu* à la fin du scénario (informations enregistrées, etc.)
